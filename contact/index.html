Contact

首版保持静态联系方式，先保证稳定和真实

这里不接在线表单，也不接第三方消息服务。等你准备好公开信息后，只需要把邮箱、GitHub 或社交链接填进数据文件即可。

邮箱

待补充常用邮箱。建议优先公开一个长期可用的联系地址。

GitHub

待补充 GitHub 主页链接。公开代码仓库有助于增强可信度。

社交链接

如果暂时没有其他公开账号，可以先保留空状态，不影响首版上线。

补充建议

准备公开联系方式时，可以按这个顺序来

  • 建议优先补充一个常用邮箱地址
  • 如果有 GitHub 主页，可以一并公开
  • 如果还没有简历，也可以先把项目介绍写清楚

说明

为什么首版不做联系表单

静态联系方式更稳定，部署更简单，也不需要数据库、邮件服务或防垃圾内容策略。

对于求职向个人站来说，公开一个常用邮箱和 GitHub 链接，通常已经足够支撑首轮沟通。

如果后续你确实需要在线表单，可以在二期接入表单服务，但不建议把这件事阻塞首版上线。
