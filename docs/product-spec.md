# 产品与信息架构说明

## 1. 产品定位

- 网站类型：个人作品集网站
- 主要目标：面向全栈开发求职场景，展示真实项目、技能结构和成长路径
- 语言策略：中文为主
- 视觉方向：温和亲和、可信、轻个人感，避免模板化科技蓝商务风

## 2. v1 成功标准

- 页面结构完整：首页、关于我、作品集、项目详情、联系我全部可访问
- 内容结构完整：站点信息、个人信息、项目内容都可通过文件维护
- 空状态完整：资料不足时页面仍然清晰、可读、可继续迭代
- 部署完整：提交到 GitHub 后能自动构建并发布到 GitHub Pages
- 协作完整：其他智能体能直接依据文档继续补内容或做迭代

## 3. 页面职责

### 首页 `/`

- 提供第一印象和整体定位
- 展示目标方向、内容策略和精选项目入口
- 引导访客进入作品集或关于我页面

### 关于我 `/about`

- 承载个人介绍、技能分组、经历与教育信息
- 在内容不足时保留结构和优雅空状态

### 作品集 `/projects`

- 展示全部公开项目
- 项目卡片负责概览，详情页负责深度说明

### 项目详情 `/projects/[slug]`

- 固定展示：项目简介、角色、技术栈、关键难点、解决方案、结果与收获、外部链接
- 如果 Markdown 正文有补充内容，则继续展示扩展说明

### 联系我 `/contact`

- 只放静态联系方式
- 不接表单，不接第三方消息服务

## 4. 设计与内容原则

- 真实优先：不使用虚构个人经历、虚构公司或虚构项目
- 结构先行：先保证信息架构和展示体验，再逐步补足内容
- 项目导向：优先讲清楚做过什么、怎么做、解决了什么问题
- 易维护：新增内容应尽量只改数据或 Markdown 文件

## 5. 组件与实现边界

- 页面组件可以继续扩展，但不应破坏现有内容驱动模式
- 新增项目时不应修改核心模板逻辑
- 二期功能暂不实现：CMS、自定义域名、博客、在线表单、多语言、项目筛选搜索
